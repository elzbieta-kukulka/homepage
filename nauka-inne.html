Jak działa Internet i przeglądarka

Komunikacja przeglądarka-serwer odbywa się na zasadach request-response

Protokół, który obsługuje strony internetowe i ich zawartość to http (aktualnie coraz częściej HTTP/2).
HTTP - HyperText Transfer Protocol - to protokół do transportowania htmla (i całej reszty, którą widzisz na stronach

HTTPS - Hypertext Transfer Protocol Secure = szyfrowany, bezpieczny

URL - Uniform Resource Locator

Kody odpowiedzi:

  • 1xx - hold on
  • 2xx - here you go
  • 3xx - go away - szukasz w złym miejscu - przekieruję Cię w dobre. 301-przekierowania
  • 4xx - you fucked up - 404-nie mam tego czego potrzebujesz
  • 5xx - I fucked up - błąd wewnętrzny serwera

Czym jest root - /(slash)

w którym miejscu przzeglądarka powinna szukać zasobu.

--> ścieżka bezwględna src="/img/wtf.svg"

--> ścieżka względna bez slasha src="img/wtf.svg"

Brak / z przodu drugiej ścieżki uczynił adres do zasobu relatywnym, dlatego przeglądarka szukała pliku w aktualnym "katalogu"

Terminal

Otwarcie - Menu Start - wpisujemy cdm - wiersz polecenia

  • dir - pokazuje wszystko co znajduje się w tym katalogu
  • cd C:\ - wejdź do dysku C (cd=change directory)
  • cd d+tab - pokazuje w co mogę wejść (np d jak dev)
  • mkdir - utwórz katalog
  • code . - uruchom Visual Studio Code w tym katalogu
  • cd .. - przejście do katalogu wyżej
  • cd - wyświetlenie aktualnej ścieżki

GIT

Git składa się z repozytorium, czyli projekt, czyli wszystkie pliki. Komendy:

  • git --version - wersja
  • konfiguracja naszego gita, tylko raz, można poprawić nadpisując - git config --global user.name "Elzbieta Kukulka",
    git config --global user.email elzbietakukulka4@gmail.com
  • git config --get user.name lub user.mail - sprawdzenie
  • git init - po przejściu do projektu, w którym chcemy załoożyć repozutorium i wpisujemy git init
  • commit = stan projektu i opis tego co się zmieniło od ostatniego zapisu
    1. git status - podejrzenie
    2. git add . - dodaje wszystkie pliki do śledzenia
    3. git commit -m "opis zmian", np "initial commit"
    4. gitk - historia wszystkich zmian

GitHub

Portal, który pozwala trzymać repozytorium poza komputerem. Zakładanie repozytorium - + new repository, wpisujemy nazwę, reszta bez zmian, create, kopiujemy do terminala dwie komendy.

Wrzucanie kodu na GitHuba:
git add . / git commit -m "opis zmian" / git push
